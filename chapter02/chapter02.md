Text -> split/Tokenize -> Token IDs -> Token embeddings + Positional embeddings -> Input embeddings

作为输入处理流水线的一部分，输入文本首先被分解为单个的 token。然后使用词表将这些 token 转换为 token ID。 Token ID 被转换为嵌入向量，向其中添加类似大小的位置嵌入后，就产生了作为主要 LLM 层输入的“输入嵌入”。

| 问题 (Question) | 回答 (Answer) |
| :--- | :--- |
| **为什么要把文本转换成数字 ID (Token)？** | 计算机无法处理字符，只能运算数字。Tokenization 为每个词分配“身份证号”，将其转化为离散的整数索引。🔢 |
| **为什么有了 ID 还要转换成向量 (Embedding)？** | 整数 ID 本身不包含语义联系。Embedding 将 ID 映射到空间坐标，使得**语义相近**的词在空间中距离更近，捕捉词汇间的逻辑关联。📍 |
| **为什么维度一定要非常高（如 768 维），3 或 4 维不行吗？** | 人类语言极其复杂。一个词通常包含多种属性（如：苹果既是水果也是科技公司）。低维度空间无法同时满足多个语义维度的排列要求，会导致**语义冲突**或**歧义**。多维度提供了足够的“余地”来容纳复杂的词汇关系。🌌 |
|**为什么要用Position embedding**|因为模型看到的token是数学上的集合，没有顺序关系，它只计算token之间的相似度，不考虑第几个词/前后关系/顺序编号。把“位置”变成向量，加进 token embedding 里可以让模型强制考虑相对关系。考虑绝对举例->token之间的距离|


* **可以添加诸如 `<|unk|>` 和 `<|endoftext|>` 之类的特殊 token**，以增强模型的理解能力并处理各种上下文，例如处理未知单词或标记不相关文本之间的边界。  
- **用于 GPT-2 和 GPT-3 等 LLM 的字节对编码 (BPE) 分词器**，可以通过将未知词分解为子词单元或单个字符，从而高效地处理它们。   
- 我们在 token 化后的数据上**使用滑动窗口方法，来生成用于 LLM 训练的输入-目标对**。  
* ***PyTorch 中的嵌入层充当查找操作**，它检索与 token ID 相对应的向量。生成的嵌入向量提供了 token 的连续表示，这对于训练像 LLM 这样的深度学习模型至关重要。
* 虽然 token 嵌入为每个 token 提供了保持一致的向量表示，但**它们缺乏 token 在序列中位置的概念**。为了弥补这一点，存在两种主要类型的位置嵌入：绝对的（absolute）和相对的（relative）。OpenAI 的 GPT 模型利用绝对位置嵌入，它们被添加到 token 嵌入向量中，并在模型训练期间不断进行优化。
